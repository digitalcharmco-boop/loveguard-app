## Step 1: Push to GitHub

### Create the GitHub repo
1. Go to https://github.com/new
2. Name it: `loveguard-app`
3. Set to **Private** (you have live Stripe keys in the app logic)
4. **Do NOT** initialize with README (repo already has files)
5. Click **Create repository**

### Push from your local machine
Open PowerShell or Command Prompt (or WSL) in `C:\Users\charm\loveguard-app\`:

```bash
cd /mnt/c/Users/charm/loveguard-app   # WSL path

# Add your GitHub remote (replace YOUR_USERNAME with your GitHub username)
git remote add origin https://github.com/YOUR_USERNAME/loveguard-app.git

# Push
git push -u origin main
```

If prompted, authenticate with your GitHub credentials or a Personal Access Token.

---

## Step 2: Deploy on Streamlit Cloud

1. Go to https://share.streamlit.io
2. Sign in with GitHub (use the same account that owns the repo)
3. Click **"New app"**
4. Fill in:
   - **Repository:** `YOUR_USERNAME/loveguard-app`
   - **Branch:** `main`
   - **Main file path:** `app.py`
5. Click **"Advanced settings"** → **Secrets** tab
6. Paste the following (fill in YOUR real keys):

```toml
OPENAI_API_KEY = "sk-..."
STRIPE_SECRET_KEY = "sk_live_..."
STRIPE_PUBLISHABLE_KEY = "pk_live_..."
APP_ENV = "production"
DEBUG = "False"
GOOGLE_CLOUD_PROJECT = "instant-ranking-418806"
```

7. Click **"Deploy!"**

Your app will be live at:
```
https://YOUR_USERNAME-loveguard-app-app-XXXXX.streamlit.app
```

## ⚠️ Important Notes

- **NEVER commit `.env`** — it's gitignored. Real keys go in Streamlit Cloud Secrets only.
- **`google_credentials.json`** is excluded from git (contains OAuth client secret). You don't need it for Streamlit Cloud deployment — it's only for Google Cloud SDK auth.
- **Payment flows** currently show a "Demo mode" success message — the Stripe integration creates real payment intents but there's no frontend payment form. To enable real payments, add Stripe.js + a webhook endpoint (future enhancement).
- Free tier: 3 analyses per session. Premium: unlimited (upgrade button shown after limit).

## Troubleshooting

| Problem | Fix |
|---|---|
| `ModuleNotFoundError: openai` | Check requirements.txt has `openai>=1.0.0` ✅ already fixed |
| `AttributeError: module 'openai' has no attribute 'OpenAI'` | Old openai installed — requirements.txt now pins v1.x |
| `StreamlitAPIException: st.experimental_rerun` | Already fixed → `st.rerun()` |
| App can't find secrets | Make sure you added them in Streamlit Cloud → Settings → Secrets |
| Stripe errors | Normal — payments need a real frontend form to complete |
